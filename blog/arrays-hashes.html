Arrays and Hashes

2/14/16



Arrays and Hashes are both where data is stored. But they are a bit different, and I'll outline them below.

Arrays
Arrays are an ordered list of elements that can store strings, integers, and/or floats. You can put whatever data you want in there, they don't all have to be strings and they don't all have to be integers; they can be a combination. Each piece of data in the array is assigned an index number. They look like this:



The first item on that list is index numer 0, then the next is 1, and the one after that is 2.

Hashes
Hashes are a little different. Values in Hashes are stored in key-value pairs. It's a lot more organized than an array, and is a more organized way to store data. Items in an array can be unrelated, but in Hashes, they're grouped in categories. Let me just show you an example:

my_friends_ages = {
  "Brandi" => 31
  "Sarah" => 20
  "Camilla" => 18
  "Stephanie" => 27
}

Each key (my friends' names) have a corresponding value (ages).


That's all I have time for right now, but I hope that shows you what the difference between an array and a hash is and what they could be useful for.
