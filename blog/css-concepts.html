Phase 0 - Week 3

CSS Concepts

2/8/16



There are a lot of concepts to grasp in CSS, but for the sake of this blog post I'm just going to focus on one, and that is positioning.

What are the differences between relative, absolute, fixed, and static positioning?


  • Relative
      • Relative is a bit confusing. If you set something to relative it might appear to do nothing. That's because you have to tell it where to go by specifying top:__, bottom:__, left:__, and right:__. And then the element will be positioned based on what you specify. So if you put top:50px, then the element will move 50px off from the top, so basically, it will move 50px down. If you have static elements on the page, relative can move on top of the static element while the static element stays in the same spot.
  • Absolute
      • Absolute positioning is a bit different. So, relatively positioned elements don't affect static elements at all. However, if you set an element to be absolute, it's like it doesn't exist, and a static element can move to fill the spot that the absolute element would have been. Absolute positions are also determined by its parent elements.
  • Fixed
      • Fixed elements arecompletely independent and they will be positioned based on the browser window. Also, when the page is scrolled, the element will stay where it is and not follow along when you scroll.
  • Static
      • Static is the "default" positioning, so when you don't specifiy a position, it will automatically be set to static. Elements that are specified as static will stack one after the other in the order that they come in your code.

    I hope that makes sense, if not, don't worry, I still don't fully understand it either. But with practice and research, it will start to make more sense to you.
