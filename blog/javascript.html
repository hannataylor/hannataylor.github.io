Phase 0 - Week 7

Ruby vs. JavaScript

3/7/16



This week I started learning JavaScript. In learning JavaScript, I quickly found that many things in the JavaScript language are similar to Ruby. One of those being JavaScript objects. JavaScript objects are much like Ruby Hashes. Both store data, in an organized way with keys and values.

I'll show you a hash in Ruby and an object in JavaScript. Take note of the differences, and then I'll explain the differences below.

Ruby Hash

city = {
name: "New York City",
state: "New York",
population: "8.406 million",
boroughs: ["Brooklyn","Queens","Manhattan","the Bronx","Staten Island"],
mayor: "Bill de Blasio"
}

JavaScript Object

var city = {
name: "New York City",
state: "New York",
population: "8.406 million",
boroughs: ["Brooklyn","Queens","Manhattan","the Bronx","Staten Island"],
mayor: "Bill de Blasio"
};

Notice any differences? In JavaScript, variables always have to be defined by using the keyword var in front of the variable you want to define. Then to close the hash, JavaScript has slightly different syntax, and you just have to put a semicolon after the closing curly brace

Now, how about if we want to access an item in the hash/object? What are the differences?

Ruby Hash

# to access within your code
city[:population]
=>"8.406 million"

# to print on screen
puts "#{city[:name]} has a population of #{city[:population]} people."
=>New York City has a population of 8.406 million people

JavaScript Object

# to access within your code
city.population;
=>"8.406 million"

# to print on screen
console.log(city.name + " has a population of " + city.population + " people.")
=>New York City has a population of 8.406 million people

Do you see the minor differences?
In Ruby you access the key/value pair like this: hashname[:key]
In JavaScript, you simply write it like this: hashname.key;

Simple as that! The biggest challenge in moving from working in Ruby to JavaScript is learning the differences in syntax. JavaScript has a lot of curly braces {} and semicolons ; and it make take some getting used to. But otherwise, creating things like a hash/object are more or less the same, minus a few minor syntactical differences.
