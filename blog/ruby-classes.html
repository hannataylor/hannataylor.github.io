Phase 0 - Week 6

Ruby Classes

2/26/16



Today I'm going to teach you a little something about Classes in Ruby.

You might've heard the term "Object-Oriented Programming" before. Well, Ruby is an example of "Object-Oriented Programming". Everything in Ruby is an object. And these objects can take methods the things that the method tells them to do. And again, as you already know you can define variables, and create arrays and hashes and create all sorts of objects.

What is a class? A class is sort of like a blueprint for creating objects. It'll help you organize all those little variables and methods you have, and will keep them contained within the class. I think explaining classes with words just doesn't get the concept across, especially if this is your first time learning about them. I think it's best to look at code, and see what it's doing.

I just moved to New York City last week, and I'm starting to miss my dogs back home, so I'm going to use dogs as an example. I'm going to create a class called "Dog". And it's going to be all about dogs.

Class Dog
def initialize (breed,name,sex,age,toy)
@breed = breed
@name = name
@sex = sex
@age = age
@toy = toy
end

def bark
p "Woof! Woof!"
end

def gender
if @sex == "female"
return "Her"
elsif @sex == "male"
return "His"
else
return "Its"
end

end
def display
p "#{@name} is a(n) #{@age} year old #{@breed}. #{gender} favorite toy is a #{@toy}"
end
end

new_dog = Dog.new("Yorkie","Sawyer","male","8","Squirrel")

new_dog.display
new_dog.bark

#Your ruby interpreter will display:
"Sawyer is a(n) 8 year old Yorkie. His favorite toy is a Squirrel"
Woof! Woof!

Did you follow along with what was happening?

• First, I defined the class. For example this class was Dog. I could make another class called Cat. And another class called Car, or anything else. Do you see how putting things in classes would keep things organized? You wouldn't want your variables conflicting with another. And you wouldn't want to accidentally change a variable when you're working on a group of objects that are unrelated.

• Then the first method you have to make is the initialize method. This sets up the class with some variables to get you started.

• Then from there you can define whatever methods you want.

• After you finish adding your methods, make sure you end the class.

• Then to use your class you can start a new object of that class. In this case, I named it new_dog, and you create the new object like this:

new_dog = Dog.new("Yorkie","Sawyer","male","8","Squirrel").

When you initialized it, you allowed certain variables as objects. When you create the new object, you have to fill in arguements with the correct number of variables.

• Once the new object is made, you can call the different methods you made, it Ruby will run the program you created.


Summary

I hope that makes sense, after looking at the example code I made for you. Classes are like blueprints for objects and keeps your methods and variables contained within the class, so when you go on to a different subject or just different things in the code, you can keep them all organized within classes to prevent accidentally creating bugs in your code.
