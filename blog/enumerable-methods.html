Enumerable Methods

2/21/16



Today I'm going to talk about Enumerables, and in particular, one Enumerable method: Enumerable#cycle.

What is an Enumerable?

The dictionary definition of "enumerate" is: to mention (a number of things) one by one.

Enumerable is a module. What's a module? An example of modules are Arrays and Hashes. Modules have methods that you can call on the module. You've seen methods before; some methods you've probably seen before are: .each, .times, .length, etc...Those sorts of things.

So since Enumerable is a module it also has methods you can use. Enumerables are also called mix-ins, because you can mix-in its methods with other modules like arrays and hashes.

Basically, how I see it, enumerables are like bonus methods that make your life easier. So back to the dictionary definition of "enumerate". The definition says it mentions things one by one, well that's kind of what enumerables do. For example with arrays, you've probably done a lot of .each do |x| type things. But with enumerables, it goes through things one by one for you. I'll explain what I mean when I teach you about Enumerable#cycle next.

Enumerable#cycle

.cycle is just one example of an enumerable method that I'm going to show you. So you know how you can print a string multiple times by doing:

string = "Hello, world!"
5.times do print string
end

And your Ruby interpreter will print out the phrase "Hello, world!" 5 times? Well with .cycle, we can basically do the same thing, but in an easier way.

Using .cycle, we can say how many times we want to do what we want the element to do.

Here's an example:

string = "Hello, world!"
string.cycle(5) { |x| puts x}

That bit of code there would print "Hello, world!" 5 times.

Or if we did it like this:

string = "Hello, world!"
string.cycle { |x| puts x}

.cycle would tell the Ruby interpreter to keep printing string FOREVER! So it works like a loop, too.

At first it doesn't seem like it would be helpful. But let's say you weren't sure how many times you needed to print "Hello, world!" Let's say at first you think you might need to print it 10 times, so you use the 10.times do method. Seems easy enough. Not sure why you'd want to print "Hello, world!" repeatedly, but for the sake of this example, that's what you want to do. What if later you change your mind and you decide you need "Hello, world!" to print forever, until the end of time. Well, if you want to do that, you're going to have to write a whole new method. What a waste of time?! Right? Well, if you just used the enumerable method .cycle, from the beginning, you could change it to cycle and do what you ask it to do easily. You could make it cylce forever, or you could make it cycle a set number of times. Either way, it's one method to do both loop it and/or print it a set number of times.

In conclusion, enumerable methods are kind of like easy shortcuts to make your life easier and you should use them when you can.
